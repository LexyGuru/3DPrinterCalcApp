# 📋 История версий - 3DPrinterCalcApp

Этот документ содержит подробный журнал изменений для всех версий приложения 3D Printer Calculator.

## v3.0.0 (2025) - 🔒 Шифрование Данных Клиентов & Соответствие GDPR

### 🔒 Шифрование Данных Клиентов
- **Шифрование AES-256-GCM** - Зашифрованное хранение данных клиентов с использованием отраслевого стандартного алгоритма AES-256-GCM
- **Хеширование паролей PBKDF2** - Безопасное хранение паролей с использованием алгоритма PBKDF2 (100 000 итераций, SHA-256)
- **Хранение в отдельном файле** - Зашифрованные данные клиентов хранятся в отдельном файле `customers.json`
- **Управление паролями в памяти** - Пароли хранятся только в памяти и удаляются при закрытии приложения
- **Интеграция пароля приложения** - Опционально: пароль защиты приложения также может использоваться для шифрования
- **Система запроса пароля** - Интеллектуальный запрос пароля (не отображается на экране загрузки, после приветственного сообщения)
- **Защита целостности данных** - Зашифрованные данные защищены от случайного удаления

### ✅ Защита Данных в Соответствии с GDPR/ЕС
- **Соответствие**: Приложение обрабатывает данные клиентов в соответствии с GDPR (Общий регламент по защите данных) и правилами защиты данных ЕС
- **Отраслевое стандартное шифрование**: Использование алгоритма AES-256-GCM (соответствует рекомендациям ЕС)
- **Безопасное управление паролями**: Алгоритм хеширования PBKDF2 (рекомендуется NIST)
- **Минимальный сбор данных**: Хранит только необходимые данные клиентов, требуемые приложением
- **Хранение данных**: Пользователь имеет полный контроль над хранением и удалением данных
- **Контроль доступа**: Доступ, защищенный паролем, к данным клиентов

### 🎨 Улучшения UI/UX
- **Модальное окно активации шифрования** - Новое модальное диалоговое окно для активации шифрования с опцией пароля приложения
- **Расширение ConfirmDialog** - Поддержка пользовательского содержимого для модальных компонентов
- **Время запроса пароля** - Интеллектуальное отображение (не на экране загрузки)
- **Интеграция настроек** - Настройки шифрования на вкладке Безопасность

### 🔧 Технические Улучшения
- **Модуль шифрования backend** - Шифрование, реализованное на Rust (`src-tauri/src/encryption.rs`)
- **Утилиты шифрования frontend** - Вспомогательные функции TypeScript для управления шифрованием
- **Менеджер паролей** - Система управления паролями в памяти
- **Интеграция хранилища** - Функции saveCustomers/loadCustomers с интеграцией шифрования

### 📚 Языковая Поддержка
- **13 языков обновлено** - Новые ключи перевода шифрования во всех языковых файлах
- **Новые ключи**: `encryption.enableEncryption`, `encryption.useSamePasswordAsApp`, `encryption.encryptionPassword`, `encryption.passwordMinLength`

## v2.0.0 (2025) - 🚀 Мониторинг производительности & Система аудитного журнала

### 🌐 Локализация Сообщений Консоли
- **Полная локализация консоли** - Все сообщения консоли отображаются на выбранном языке
- **Перевод операций хранилища** - Сообщения загрузки и сохранения (принтеры, филаменты, настройки, предложения, клиенты, проекты, задачи)
- **Перевод сообщений резервного копирования** - Ежедневная проверка резервного копирования, создание резервной копии, сообщения ротации
- **Перевод сообщений ротации журналов** - Сообщения ротации журналов и журналов аудита с динамическими частями
- **Перевод метрик производительности** - Метрики CPU и памяти, регулярные сообщения журнала
- **Перевод системных сообщений** - Инициализация приложения, инициализация журнала frontend, приветственное сообщение
- **Перевод многочастных сообщений** - Перевод частей данных сообщений консоли (дата, метка времени, файл, информация о статусе)
- **Поддержка 13 языков** - Все сообщения консоли переведены на английский, венгерский, немецкий, испанский, итальянский, польский, португальский, русский, украинский, чешский, словацкий и китайский

### ⚡ Логирование метрик производительности
- **Класс Performance Timer** - Ручное измерение времени операций
- **Измерение времени загрузки** - Записаны все времена загрузки модулей (Settings, Printers, Filaments, Offers, Customers)
- **Измерение времени операции** - Автоматическое измерение времени для асинхронных и синхронных операций
- **Мониторинг использования памяти** - Отслеживание и логирование памяти кучи JavaScript
- **Мониторинг использования CPU** - Регулярное измерение использования CPU каждые 5 минут
- **Сводка производительности** - Агрегированная статистика для времен загрузки и операций
- **Структурированные сообщения журнала** - Подробное отображение процента CPU и значений памяти
- **Команды производительности backend** - Команда backend `get_performance_metrics` для данных CPU и памяти

### 🔐 Реализация аудитного журнала
- **Инфраструктура аудитного журнала** - Отдельный файл аудитного журнала (`audit-YYYY-MM-DD.json`)
- **Логирование критических операций**:
  - CRUD операции (Создание/Обновление/Удаление для Filaments, Printers, Offers, Customers)
  - Изменения настроек (тема, язык, настройки журнала, автосохранение, и т.д.)
  - Операции резервного копирования (создание, восстановление)
  - Операции сброса к заводским настройкам
  - Запись ошибок
- **Просмотрщик аудитного журнала** - Виртуальная прокрутка для больших файлов, с фильтрацией, поиском и возможностями экспорта
- **Автоматическая очистка** - Старые файлы аудитного журнала автоматически удаляются на основе настроенных дней хранения
- **Команды backend** - `write_audit_log`, `list_audit_logs`, `read_audit_log_file`, `delete_old_audit_logs`
- **Полная локализация** - Все 13 поддерживаемых языков

### 🎯 Улучшения UI/UX
- **История аудитного журнала** - Двухколоночный макет в разделе Настройки → Управление журналами
- **Отображение метрик производительности** - В модальном окне диагностики системы
- **Обновления просмотрщика журналов в реальном времени** - Переключатель автоматического обновления, обнаружение изменений на основе хеша
- **Уточнение автоматической прокрутки** - Осознание позиции прокрутки пользователя

### 🔧 Технические улучшения
- **Оптимизация проверки обновлений GitHub** - При запуске и каждые 5 часов (на основе localStorage)
- **Формат тега beta** - Отдельный тег `beta-v2.0.0` для beta-версий (не перезаписывает основную версию)
- **Логика проверки версии** - Поиск beta-версии на основе префикса `beta-v`

## v1.7.0 (2025) - 💾 Система резервного копирования, экран загрузки и улучшения библиотеки филаментов

### 💾 Полная реализация системы резервного копирования
- **Автоматическая система резервного копирования** - Один файл резервной копии в день (создается только в новый день)
- **Хук напоминания о резервном копировании и компонент UI** - Уведомление, если резервной копии нет
- **UI истории резервного копирования в Настройках** - Список с цветовым кодированием (зеленый/желтый/красный/серый) для возраста файла резервной копии и обратный отсчет удаления
- **Модальное окно автосохранения** - Объяснение, когда автосохранение включено
- **Синхронизация автосохранения и автоматического резервного копирования** - Автоматическое резервное копирование при сохранении с автосохранением
- **Сброс к заводским настройкам с автоматическим удалением файлов резервной копии**
- **История резервного копирования автоматически обновляется**, когда автосохранение включено

### 🔧 Оптимизация backend системы резервного копирования
- **Добавлены команды backend** для удаления старых резервных копий (`cleanup_old_backups_by_days`, `cleanup_old_backups_by_count`)
- **Функции очистки frontend обновлены для использования команд backend**, устраняя ошибки "forbidden path"
- **Все операции с файлами (создание, удаление, перечисление) теперь происходят из backend**, избегая проблем разрешений Tauri

### ⚡ Оптимизация производительности системы резервного копирования
- `hasTodayBackup()` оптимизировано: использует команду backend `list_backup_files`, не нужно читать все файлы
- **Добавлен механизм блокировки** для предотвращения параллельных резервных копий
- **Быстрее работа** даже при большом количестве файлов резервных копий

### 📁 Открытие каталога резервного копирования и история журналов
